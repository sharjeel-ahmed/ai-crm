Pazo CRM API

Complete REST API for the Pazo CRM platform. Access contacts, companies, deals, activities, reports, and more. Supports both JWT bearer tokens for web applications and API keys for external integrations and mobile apps.

Base URL

http://localhost:3001

Internal endpoints: /api/*

Public v1 API: /api/v1/*

Response Format

All responses are JSON. Successful responses return data directly (no wrapper). Errors return { "error": "message" }

Roles

admin — full access, user management

manager — see all data, delete records

sales_rep — only own contacts/deals

Default Credentials

Email: admin@pazo.com

Password: admin123

Authentication

The API supports two authentication methods. Use JWT for the web app, and API keys for external integrations.

JWT Bearer Token

For web app & write operations. Obtain via POST /api/auth/login.

Authorization: Bearer eyJhbGciOi...

API Key (v1 endpoints)

For external integrations. Create via POST /api/api-keys (requires JWT). Read-only access.

X-API-Key: pzo_a1b2c3d4e5f6...

API keys inherit the creating user's role. A key created by a sales_rep only returns that rep's data. Admin keys return everything.

Rate Limiting

API v1 endpoints are rate-limited per API key. JWT endpoints are not rate-limited.

Limit100 requests per minute per API key
Header: X-RateLimit-LimitMaximum requests in window
Header: X-RateLimit-RemainingRemaining requests in current window
Header: X-RateLimit-ResetUnix timestamp when window resets
Exceeded response429 Too Many Requests with Retry-After header

Error Handling

Errors return a JSON object with an error field and an appropriate HTTP status code.

{
  "error": "No token provided"
}
StatusMeaning
400Bad request — missing or invalid parameters
401Unauthorized — missing or invalid authentication
403Forbidden — insufficient role permissions
404Not found — resource does not exist
429Rate limited — too many requests
500Server error

API Key Management

Create, list, and revoke API keys. All endpoints require JWT authentication.

▶ POST /api/api-keys Create a new API key JWT

Request Body

FieldTypeRequiredDescription
namestringrequiredHuman-readable label for the key (e.g. "Zapier Integration")

Example Request

POST /api/api-keys
Authorization: Bearer eyJhbGciOi...
Content-Type: application/json

{
  "name": "Mobile App Key"
}

Response 201 Created

{
  "id": 1,
  "key": "pzo_a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4",
  "name": "Mobile App Key",
  "prefix": "pzo_a1b2",
  "created_at": "2026-03-15T10:30:00.000Z"
}

The full key value is shown only once. Store it securely.

▶ GET /api/api-keys List all API keys for the current user JWT

Response 200 OK

[
  {
    "id": 1,
    "name": "Mobile App Key",
    "key_prefix": "pzo_a1b2",
    "is_active": 1,
    "last_used_at": "2026-03-15T12:00:00.000Z",
    "created_at": "2026-03-15T10:30:00.000Z"
  }

Admins see all keys. Other users see only their own.

Reports

Analytics and reporting endpoints. All support date range filtering. Available on both JWT and API key (read-only).

Date Range Parameters (all report endpoints)

ParamTypeDescription
presetstringlast_week, last_month, last_quarter
start_datedateCustom range start (YYYY-MM-DD). Overrides preset.
end_datedateCustom range end (YYYY-MM-DD)
▶ GET /api/reports/dashboard Main dashboard metrics JWT API Key

Returns: snapshot (totals), movement (new/won/lost in period), stage health, top attention deals, and rep leaderboard. Sales reps see only their own metrics.

▶ GET /api/reports/funnel-dashboard Funnel, forecast, conversion rates JWT API Key

Returns: sales forecast, conversion funnel, lost deal breakdown by reason, and quota tracking per rep.

▶ GET /api/reports/summary Deals summary counts JWT API Key

Returns counts and values for: won, lost, active, stale (no activity 7+ days), and inactive deals.

▶ GET /api/reports/pipeline-value Pipeline value by stage JWT API Key

Returns total deal value grouped by pipeline stage.

▶ GET /api/reports/rep-performance Per-rep performance metrics JWT API Key

Returns per-rep: won deals count, total won value, average cycle time, and open pipeline value.

▶ GET /api/reports/deal-aging Days in current stage per deal JWT API Key

Returns each open deal with the number of days it has been in its current stage.

▶ GET /api/reports/attention Top deals needing attention JWT API Key

Returns top 10 deals ranked by attention score. Score factors: negative sentiment (+50), no recent activity (+25), days stuck in stage (up to +30).
